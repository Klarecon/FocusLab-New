0 How to read this plan

Every Key Result has a number and a date, so evaluation is binary: hit or miss, then why / why not. Two things to know before the numbers:

Commit vs. Aspire. COMMIT objectives (July / launch) must hit 100% — there's no funnel to test if the money path doesn't ship. ASPIRE objectives (Aug / Sep) are real targets built on cold-traffic assumptions no one can promise; hitting 0.7 on these is a genuinely good quarter, not a shortfall.
Numbers tagged CONFIRM are not locked. They rest on assumptions only you and Oren can set — ad budget, realistic conversion %, MRR target. I've given a recommended number + rationale for each; the point of this doc is to decide them together, not inherit them. All of them are collected in the box at the bottom.
One honesty flag that shapes September: a 30-day blended conversion metric on a funnel test that starts Sep 8 cannot fully resolve inside September — the clean verdict lands ~Oct 15. September's KRs are scoped to what's genuinely measurable by Sep 30 (run the test cleanly, get a directional read), with the definitive answer noted as arriving after quarter-end.

5 How we score it (built for yes/no + why)

Per-KR score (0.0 – 1.0)

  • Binary KRs (all of Objective 1, plus 3.1/3.3/3.4) → 1.0 if met, 0.0 if not. Yes or no.
  • Numeric KRs (all of Objective 2, plus 3.2/3.5) → score = actual ÷ target, capped at 1.0. Always report the raw number too — the headline stays yes/no, the ratio adds nuance for the debrief.

Roll-up

  • Objective score = average of its KRs. Quarter score = average of the three objectives. (Keep it equal-weight for the first quarter.)

Green / Amber / Red — differs by objective type

Objective 1 (COMMIT)
Green = 1.0. Anything under 1.0 is Red — a missed commit. No amber on launch-critical work.
Objectives 2 & 3 (ASPIRE)
Green ≥ 0.7 · Amber 0.4–0.69 · Red < 0.4. A 0.7 on a stretch goal is a win — say it out loud.
The one rule that keeps it honest: a KR only scores >0 if it's backed by a number pulled from analytics/billing on the due date — not a vibe. If we can't measure it, it scores 0 and we fix the instrumentation.

7 Risks to these OKRs

  • Launch traffic underperforms the 1,500 assumption. Every Objective-2 number scales off it. Mitigation: line up a second Reddit/community wave and a "share your result" loop before launch; treat 8 customers (the floor) as the real commit.
  • The build slips past late July. Supabase + billing + paywall + legal is a real 2.5–3 week solo stack; one integration (auth edge cases, webhook reliability) can eat a week. Mitigation: KR 1.1's scripted sandbox test is the go/no-go gate — if it's not green by Jul 25, protect launch by cutting return-loop polish, not by shipping a broken money path.
  • The September budget can't produce a real verdict. ~$1,500 gives a directional read but too few paid conversions to call a winner on economics, and the 30-day window bleeds into October. Mitigation: set expectations now — September = directional call + clean instrumentation, confirmed verdict Oct 15.
  • Cold economics simply don't close in one quarter. CAC may exceed LTV on every channel — normal for a $29 tool at launch, and a finding, not a failure. Mitigation: KR 3.4 explicitly allows "ruled out with evidence" as a pass.
